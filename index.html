1.1

Lo más importante, y qué se puede modificar

1

Lectura del Excel (codigos_NCR.xlsx)

El bot lee 4 columnas exactas del archivo y arma una lista por cada una: el código a buscar, el nuevo código de referencia, la variable de promoción y su texto.

No cambiar los nombres de columna
2

El bucle principal (for cod_redencion in cod_redenciones)

Repite todo el proceso de edición una vez por cada fila del Excel, en el mismo orden en que aparecen.

No tocar salvo que cambie el flujo de la página
3

Valores fijos de negocio: "100" y "15"

"100" se escribe siempre en "Actualización de Variable Promo"; "15" es el límite de usos fijo en cada promoción.

Modificar si cambia la regla de negocio
4

El interruptor Cancelar / Guardar (modo prueba)

Ahora mismo cada vuelta termina en "Cancelar" para no aplicar cambios reales. El bloque de código de arriba deja comentado exactamente qué reemplazar para pasar a "Guardar".

Modificar cuando se pase a modo real
5

Tiempos de espera (time, time2)

Pausas fijas (1s) y tiempo máximo de espera (20s) para que la página reaccione.

Aumentar si la página va lenta y el bot falla por timeout
6

Los XPath (rutas dentro de la página)

Son las "direcciones" que usa el bot para encontrar cada botón o campo dentro del HTML de la página del panel NCR.

Solo si Cencosud cambia el diseño del sistema

Conceptos básicos de Python

Antes de entrar al código real, estas son las diez ideas de Python que vas a necesitar para leer todo lo que sigue.

"""
====================================================================
 BOT CRM - REDENCION NCR (version documentada para aprendizaje)
====================================================================

QUE HACE ESTE SCRIPT (en una frase):
Automatiza, dentro del navegador Chrome, la edicion repetitiva de
codigos de promocion en el panel NCR, usando como fuente de datos
un archivo Excel (codigos_NCR.xlsx).

FLUJO GENERAL:
  1. Abre Chrome y navega a la URL del panel NCR.
  2. Pide usuario/contrasena por ventana emergente e inicia sesion.
  3. Entra al modulo "Promociones".
  4. Pide el codigo de campana y la ruta del archivo de clientes.
  5. Lee el Excel "codigos_NCR.xlsx" (4 columnas clave).
  6. Por cada fila del Excel, repite un ciclo de edicion en la web:
     buscar codigo -> editar -> cambiar referencia -> configurar
     nivel/tipo de beneficio -> variable de promocion -> detalles
     adicionales -> subir archivo de clientes -> (Cancelar, modo
     prueba) o Guardar si se activa el modo real.
  7. Al final muestra cuantos procesos se ejecutaron.

NOTA IMPORTANTE:
Actualmente el script esta en MODO PRUEBA: al final de cada
iteracion presiona el boton "Cancelar" en vez de "Guardar", para
no aplicar cambios reales mientras se aprende/prueba el bot.
"""

# ====================================================================
#  CONCEPTOS BASICOS DE PYTHON QUE VAS A VER EN ESTE ARCHIVO
# ====================================================================
#
# Si nunca has programado en Python, estas ideas te van a ayudar a
# leer todo el archivo. No hace falta memorizarlas, solo tenerlas
# como referencia mientras lees el codigo de abajo.
#
# 1) VARIABLES
#    Una variable es un nombre que guarda un valor.
#      nombre = "Juan"      -> guarda un texto (string)
#      edad = 25             -> guarda un numero (int)
#      lista = [1, 2, 3]     -> guarda una lista de valores
#    No hace falta declarar el "tipo" de antemano, Python lo deduce
#    solo segun el valor que le asignes.
#
# 2) COMENTARIOS
#    Todo lo que empieza con "#" es un comentario: Python lo ignora
#    por completo, solo es texto explicativo para quien lee el
#    codigo (como este bloque que estas leyendo ahora mismo).
#    Un texto entre """ triples comillas """ tambien puede usarse
#    como comentario de varias lineas, o como una nota que "apaga"
#    un bloque de codigo completo sin borrarlo (lo veras mas abajo
#    en el codigo original, en partes que quedaron comentadas).
#
# 3) INDENTACION (sangria)
#    A diferencia de otros lenguajes que usan llaves { }, Python usa
#    los ESPACIOS AL INICIO DE LA LINEA para saber que codigo
#    pertenece a que bloque. Por ejemplo, todo lo que este "metido"
#    hacia adentro dentro de un "for" pertenece a ese bucle. Si la
#    sangria esta mal, el programa directamente no funciona.
#
# 4) IMPORT (traer herramientas hechas por otros)
#    "import selenium" trae una libreria (una caja de herramientas
#    ya escrita por otras personas) para poder usarla en este
#    archivo. "from selenium import webdriver" trae solo una pieza
#    especifica de esa libreria. El "as" le pone un apodo mas corto,
#    por ejemplo "import tkinter as tk" hace que despues se pueda
#    escribir "tk.Tk()" en vez de "tkinter.Tk()".
#
# 5) OBJETOS Y METODOS (la sintaxis con puntos)
#    En Python casi todo es un "objeto" que trae consigo acciones
#    que puede hacer, llamadas "metodos". Se llaman con un punto:
#
#        driver.get(url)
#        ^^^^^^  ^^^ ^^^
#        objeto  metodo  argumento (el dato que le pasas)
#
#    Aqui "driver" es el objeto que representa el navegador Chrome
#    controlado por el bot. ".get(url)" le dice "ve a esta pagina".
#    Los metodos se pueden encadenar uno tras otro:
#
#        driver.find_element(By.ID, 'boton').click()
#
#    Esto se lee de izquierda a derecha: "en el navegador, busca el
#    elemento con ese ID, y luego haz clic en el".
#
# 6) FUNCIONES CON PARENTESIS
#    Cualquier palabra seguida de parentesis, como sleep(1) o
#    print("hola"), es una funcion: hace algo con la informacion
#    que le pones dentro de los parentesis (sus "argumentos").
#
# 7) LISTAS Y SU INDICE (el numero de posicion)
#    Una lista guarda varios valores en orden, por ejemplo:
#
#        colores = ["rojo", "verde", "azul"]
#
#    Para sacar un valor especifico se usa su POSICION entre
#    corchetes, y OJO: en Python se empieza a contar desde el 0:
#
#        colores[0]  -> "rojo"   (el primero)
#        colores[1]  -> "verde"  (el segundo)
#        colores[2]  -> "azul"   (el tercero)
#
#    Mas abajo en este script se usa una variable llamada "cont"
#    que funciona exactamente asi: es un contador que dice "en que
#    posicion de la lista voy", y se usa como indice: por ejemplo
#    cod_promovares[cont] significa "el valor de esa lista que le
#    corresponde a la vuelta actual del bucle".
#
# 8) BUCLE "for" (repetir un bloque de codigo varias veces)
#    Sirve para repetir las mismas instrucciones una vez por cada
#    elemento de una lista:
#
#        for color in colores:
#            print(color)
#
#    Esto imprime "rojo", luego "verde", luego "azul": la variable
#    "color" va tomando, una vuelta a la vez, cada valor de la
#    lista "colores". En este bot, el "for" principal recorre la
#    lista de codigos de redencion leidos del Excel, y por cada uno
#    repite todo el proceso de edicion en la pagina web.
#
# 9) CONCATENAR TEXTOS (unir strings con "+")
#    El simbolo "+" entre textos los pega uno despues del otro:
#
#        "Hola " + "mundo"   ->   "Hola mundo"
#
#    Este script arma varios textos asi, por ejemplo para construir
#    la ruta (XPath) que busca un elemento especifico en la pagina,
#    pegando el codigo de una variable en medio de un texto fijo.
#
# 10) contador += 1  (sumar 1 a una variable)
#    Es una forma corta de escribir "cont = cont + 1": le suma 1 al
#    valor que ya tenia. Se usa al final de cada vuelta del "for"
#    para avanzar al siguiente elemento de las listas.
#
# Con estas 10 ideas deberias poder seguir el resto de los
# comentarios del archivo sin problema. Cuando aparezca algo nuevo
# y no obvio (como una funcion de una libreria externa), se explica
# en el momento, junto a esa linea.

Configuración inicial del navegador

Aquí se abre la ventana de Chrome que el bot va a controlar durante toda la ejecución, y se configuran los tiempos de espera.

# ------------------------------------------------------------------
# 2. CONFIGURACION INICIAL DEL NAVEGADOR
# ------------------------------------------------------------------

# Esto es una variable de texto (string) que guarda la direccion
# web a la que el bot va a entrar. Las comillas indican que es
# texto, no un numero ni un comando.
url = "https://test.aws.pos.pe.cencosud.net/dpc5/index.html#/admin"

# Options() crea un objeto de configuracion "vacio" para Chrome.
options = Options()
# .add_argument() es un metodo de ese objeto: le agrega una opcion
# de arranque, en este caso "abrir maximizado".
options.add_argument("--start-maximized")

# Service() sin argumentos usa el chromedriver disponible en el PATH
# (o el que Selenium Manager descargue automaticamente segun la
# version de Chrome instalada). No hace falta entender el detalle
# tecnico: solo es la pieza que permite que Python "hable" con Chrome.
service = Service()

# webdriver.Chrome(...) es la funcion que realmente ABRE una ventana
# de Chrome nueva y la deja lista para ser controlada. El resultado
# se guarda en la variable "driver": de aqui en adelante, "driver"
# ES el navegador — todo lo que quieras que Chrome haga, se lo pides
# a traves de esta variable (driver.algo()).
driver = webdriver.Chrome(service=service, options=options)

# Tiempos de espera globales (en segundos). Son numeros enteros
# (int), no texto, por eso van sin comillas.
driver.implicitly_wait(120)          # Espera implicita al buscar cualquier elemento
driver.set_page_load_timeout(120)    # Tiempo maximo para que cargue una pagina

print("Abriendo pagina...")   # print() muestra texto en la consola/terminal, solo para que quien corre el bot vea el progreso
driver.get(url)               # Le dice al navegador: entra a esta direccion web

# salen de simpledialog.

# "COD" de la lista "Campana" del panel.

    prompt="Introduce el codigo de campana:"
)

# Pide la ruta completa del archivo con la base de clientes que se
# subira mas adelante en "Importar Segmento" (ej: segmento ORO,
# PLATA o BRONCE).

    title="Ruta y Nombre del archivo",
    prompt="Introduce las ruta y nombre del archivo a subir:"
)

# su id. str(cod_campana) convierte el valor a texto por seguridad
# (aunque ya suele venir como texto desde el popup).
driver.find_element(By.CLASS_NAME, 'el-input__inner').send_keys(str(cod_campana))

# Un XPath es como una "direccion" muy detallada dentro del HTML de
# la pagina, que describe exactamente donde esta un elemento (que
# etiqueta, dentro de que otra etiqueta, en que posicion...). Se ve
# complicado pero es solo texto: una ruta, similar a una ruta de
# carpetas en el explorador de archivos, pero para partes de una
# pagina web.
ruta = '//*[@id="app"]/section/section/main/div/div/div[1]/div/div[2]/div[4]/div/div/div/div[1]/div[3]/table/tbody/tr/td[2]/div/div'

# WebDriverWait(driver, time2).until(condicion) significa: "espera
# hasta time2 segundos a que se cumpla esta condicion; si se cumple
# antes, sigue de inmediato; si nunca se cumple, lanza un error".
# EC.element_to_be_clickable(...) es la condicion "que este elemento
# ya se pueda clickear" (que ya haya cargado y este visible).
elemento = WebDriverWait(driver, time2).until(
    EC.element_to_be_clickable((By.XPATH, ruta))
)
# driver.execute_script(...) ejecuta codigo JavaScript directamente
# en la pagina. Aqui se usa para forzar el clic mediante JavaScript
# en vez de un clic "normal" de Selenium, porque a veces el elemento
# esta tapado visualmente y el clic normal fallaria.
driver.execute_script("arguments[0].click();", elemento)
print("campana seleccionada")

driver.set_page_load_timeout(2)

# ------------------------------------------------------------------
# 5. LECTURA DEL EXCEL DE CODIGOS (codigos_NCR.xlsx)
# ------------------------------------------------------------------
#
# El Excel debe tener EXACTAMENTE estas 4 columnas:
#
#   - "NCR (COD REDENCION)"       -> codigo que se busca en el panel
#   - "PROMOVAR (COD CUPON NCR)"  -> nuevo valor de "Referencia"
#   - "PROMOVAR QUEMADO (PMV QMD)"-> valor de "Variable de Promocion"
#   - "TEXTO PROMOCIONAL"         -> texto que acompana a la variable
#                                     de promocion en el desplegable
#                                     (deben coincidir EXACTO con una
#                                     opcion real del sistema, si no,
#                                     el bot fallara al buscarla)
#
# Cada lista queda "alineada" por posicion: la fila 0 de una columna
# corresponde a la fila 0 de las demas (por eso se usa el indice
# "cont" para recorrerlas juntas mas abajo).

# pd.read_excel(...) abre el archivo y lo convierte en una tabla que
# pandas entiende (un "DataFrame"), muy parecida a una hoja de
# Excel pero manejable desde Python. engine="openpyxl" le dice con
# que libreria interna debe leer el archivo .xlsx.
df = pd.read_excel("codigos_NCR.xlsx", engine="openpyxl")

# df["NOMBRE DE COLUMNA"] saca una columna completa de la tabla.
# A partir de ahi se encadenan 3 metodos, uno despues del otro:
#
#   .dropna()      -> elimina las filas vacias de esa columna
#   .astype(str)   -> convierte todos los valores a texto (por si
#                      Excel los guardo como numero)
#   .tolist()      -> convierte el resultado en una lista comun de
#                      Python (la misma idea de lista que viste en
#                      el punto 7 de la seccion de conceptos basicos)
#
# El resultado final es una lista de textos, uno por cada fila del
# Excel, guardada en cada una de estas 4 variables:
cod_redenciones = df["NCR (COD REDENCION)"].dropna().astype(str).tolist()
cod_promovares = df["PROMOVAR (COD CUPON NCR)"].dropna().astype(str).tolist()
variable_promociones = df["PROMOVAR QUEMADO (PMV QMD)"].dropna().astype(str).tolist()
texto_promocional = df["TEXTO PROMOCIONAL"].dropna().astype(str).tolist()

# [:5] es una forma de "recortar" una lista y quedarse solo con los
# primeros 5 elementos (util aqui solo para mostrar una muestra en
# consola, sin imprimir el Excel completo si tiene muchas filas).
print("Codigos de redencion:", cod_redenciones[:5])
print("Codigos promovar:", cod_promovares[:5])
print("Promovar quemado:", variable_promociones[:5])

cont = 0   # Contador: indica en que fila del Excel vamos (0 = primera fila, como viste en el punto 7)

Elegir la Variable de Promoción correcta

El paso más delicado: arma un texto exacto (código + descripción) y busca esa opción en el desplegable del sistema.

    # --- 6.7 Elegir la "Variable de Promocion" correcta ---
    #
    # Este es el paso mas delicado: se escribe el codigo (variable_promocion)
    # y el sistema muestra un desplegable con opciones tipo
    # "501 - Descuento Julio Cliente Oro". Hay que encontrar la opcion
    # cuyo texto sea EXACTAMENTE "{variable_promocion} - {texto_promo}".
    print("Configurando Variable de Promocion")
    campo = wait.until(
        EC.element_to_be_clickable(
            (By.XPATH, "//input[@placeholder='Variable de Promocion']")
        )
    )

    # De nuevo se usa "cont" como indice, ahora sobre dos listas
    # distintas, para traer los dos valores que le corresponden a
    # esta misma vuelta del for.
    variable_promocion = variable_promociones[cont]
    texto_promo = texto_promocional[cont]

    campo.click()
    campo.clear()
    campo.send_keys(str(variable_promocion))

    # Espera a que el sistema muestre al menos una opcion en el desplegable.
    wait.until(
        EC.presence_of_element_located(
            (By.XPATH, "//div[contains(@class,'el-select-dropdown')]//li")
        )
    )

    # Aqui se arma un XPath "a medida" pegando texto con "+" (punto 9
    # de los conceptos basicos): se busca, entre todas las opciones
    # del desplegable, la que contenga exactamente el texto
    # "{variable_promocion} - {texto_promo}" (por ejemplo "501 -
    # Descuento Julio Cliente Oro"). Si esa combinacion de texto no
    # existe tal cual en el sistema real, esta linea fallara con un
    # error de tiempo de espera agotado (timeout).
    opcion = wait.until(
        EC.element_to_be_clickable((
            By.XPATH,
            "//div[contains(@class,'el-select-dropdown')]//li[.//span[contains(text(),'"
            + str(variable_promocion) + " - " + texto_promo + "')]]"
        ))
    )
    sleep(time)
    opcion.click()
    sleep(time)

    # Confirmar esta seccion (Nivel/Tipo/Variable de beneficio).
    boton = wait.until(
        EC.element_to_be_clickable(
            (By.XPATH, "//button[.//span[normalize-space()='Aceptar']]")
        )
    )
    driver.execute_script("arguments[0].scrollIntoView();", boton)
    sleep(time)
    boton.click()
    print("aceptar")

Subir el archivo con la base de clientes

Fuerza a que el campo de archivo (oculto por diseño) sea visible, y le escribe la ruta del archivo indicada al inicio.

    # --- 6.9 Subir el archivo con la base de clientes (segmento) ---
    wait = WebDriverWait(driver, time2)
    print("Importando Segmento...")
    input_file = wait.until(
        EC.presence_of_element_located(
            (By.XPATH, "//span[contains(text(),'Importar Segmento')]/ancestor::button/following::input[@type='file'][1]")
        )
    )

    # r"" + texto: la "r" antes de las comillas crea un "raw string"
    # (texto en crudo), donde las barras invertidas \ de una ruta de
    # Windows no se interpretan como caracteres especiales. Sumarle
    # "" no cambia nada en la practica, es solo una forma de dejar
    # explicito que el resultado es un raw string.
    ruta_archivo = r"" + nombre_archivo

    # El input de tipo "file" esta oculto por diseno (Element UI), asi
    # que se fuerza a que sea visible antes de poder escribirle una ruta.
    # Este bloque entre comillas triples es JavaScript, no Python: se
    # le pasa como texto a driver.execute_script() para que lo corra
    # el navegador.
    driver.execute_script("""
    arguments[0].style.display = 'block';
    arguments[0].style.visibility = 'visible';
    arguments[0].style.opacity = 1;
    """, input_file)

    # send_keys() sobre un input de tipo "file" simula seleccionar
    # ese archivo desde el explorador de Windows, escribiendo su
    # ruta completa.
    input_file.send_keys(ruta_archivo)

    print("Guardando cambios...")
    sleep(time)

    # --- 6.10 MODO PRUEBA: presiona "Cancelar" en vez de "Guardar" ---

2.1

Antes de empezar

a

Instala Git

Abre PowerShell y comprueba si ya lo tienes:

# comprobar si Git ya está instalado
> git --version

Si sale un error, descárgalo desde git-scm.com/download/win e instálalo con las opciones por defecto.

b

Pide tu invitación al repositorio

El repositorio es privado. Bea debe agregarte como colaborador desde GitHub antes de que puedas descargar o subir nada. Te llegará un correo de invitación — acéptalo con tu cuenta de GitHub (créate una gratis en github.com si no tienes).

c

Descarga tu rama a tu computadora

Elige una carpeta (por ejemplo el Escritorio), abre PowerShell ahí, y corre:

# descarga solo tu rama, en una carpeta con tu nombre
> git clone -b pedro https://github.com/beatrizbravo538-wq/bot_crm.git bot_crm_pedro

La primera vez que subas algo, Git te pedirá iniciar sesión con tu cuenta de GitHub (se abre el navegador). Es normal, solo autoriza y listo.

c.1

Activa GitHub Copilot (gratis) en VS Code

Dentro de VS Code, abre el ícono de Extensiones (barra lateral izquierda), busca GitHub Copilot Chat e instálala (incluye Copilot). Al iniciar sesión con tu cuenta de GitHub, VS Code te ofrecerá activar Copilot Free — acéptalo, no pide tarjeta ni pago.

d

Abre la carpeta en VS Code con el agente

En VS Code: Archivo → Abrir carpeta… y selecciona tu carpeta (bot_crm_pedro). Luego abre el panel de Copilot Chat, cambia el modo a Agent (menú desplegable junto al cuadro de texto) y escríbele en español lo que necesitas — no hace falta que recuerdes el comando exacto de memoria.

El agente va a pedirte confirmación antes de subir cambios a GitHub o hacer algo que afecte al repositorio compartido. Si te pregunta, revisa lo que dice y confirma solo si tiene sentido.

Sigue estos pasos en este orden cada vez que te sientes a trabajar. Todo se ejecuta dentro de tu carpeta bot_crm_pedro.

2.4

Glosario rápido

repositorio
La carpeta del proyecto completa, con todo su historial de cambios. En GitHub le decimos "repo".
rama (branch)
Una copia paralela del proyecto donde puedes trabajar sin afectar a los demás. La tuya se llama como tú.
commit
Una "foto" guardada de tus cambios, con un mensaje explicando qué hiciste.
push
Subir tus commits desde tu computadora hacia GitHub.
pull
Bajar a tu computadora los cambios que ya están en GitHub.
clone
Descargar por primera vez una copia completa del repositorio a tu computadora.
el agente
GitHub Copilot Chat en modo Agente, dentro de VS Code. Le escribes en español lo que quieres hacer y él traduce eso a los comandos de Git correspondientes, pidiéndote confirmación antes de ejecutar comandos o subir cambios a GitHub.
Copilot Free
El plan gratuito de GitHub Copilot, con un número limitado de mensajes y sugerencias al mes. Suficiente para el uso normal de este proyecto.
2.5

Si algo sale mal

Me sale "fatal: not a git repository"
Significa que tu terminal no está abierta dentro de la carpeta del proyecto. Revisa que el Explorador esté mostrando bot_crm_<tu nombre> antes de abrir la terminal ahí.
git push me pide usuario y contraseña, y falla
GitHub ya no acepta contraseña normal por este medio. Deja que se abra el navegador para iniciar sesión (Git te lo ofrece automáticamente), o avísale a Bea para configurarte el acceso.
Veo un mensaje sobre "conflicto" (conflict)
Pasa cuando dos personas editaron la misma línea del mismo archivo. No lo resuelvas a ciegas — guarda tu trabajo y escríbele a Bea antes de seguir.
¿Puedo dañar el trabajo de Pedro, Hernando o Ibeth?
No mientras trabajes solo en tu propia rama. Cada quien tiene la suya, separada de las demás y de main.
No sé qué comando usar, ¿qué le escribo al agente?
Descríbele en tus palabras qué quieres lograr (ej: "quiero guardar mis cambios y subirlos" o "¿en qué rama estoy?"). No necesitas memorizar sintaxis de Git — para eso está el agente. Si el agente te pregunta algo que no entiendes antes de confirmar una acción, pregúntale a él qué significa antes de aceptar.
El agente me ofrece hacer algo que suena arriesgado (forzar, sobrescribir, borrar rama)
No lo aceptes sin avisarle a Bea primero. Nunca deberías necesitar "forzar" nada en tu trabajo diario — si el agente lo sugiere, probablemente algo salió distinto a lo esperado.
Si algo se ve muy raro o un comando lanza un error que no reconoces: detente y pregunta antes de seguir escribiendo comandos. Es más fácil arreglar algo a tiempo que deshacer varios pasos.
Cualquier duda, escríbele a Bea. Este manual cubre el uso diario — no necesitas saber nada más de Git para colaborar en bot_crm.
